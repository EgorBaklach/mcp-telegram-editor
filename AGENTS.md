# AGENTS.md — Руководство по архитектуре и протокол разработки

> Данный файл является глобальной инструкцией по разработке для ИИ-агентов в этом репозитории. Здесь зафиксированы правила взаимодействия и жесткие технические ограничения сервера.

---

## 🤝 Протокол работы (Step-by-Step & Discussion-First)

Перед внесением любых изменений в код (кроме файлов планирования/описания):
1. **Анализ идеи**: Изучить шаг, предложенный пользователем.
2. **Оценка ресурсов**: Оценить риски потребления RAM/CPU, OOM (Out of Memory) и параллельной нагрузки.
3. **Обсуждение**: Предложить оптимизированное решение и обсудить его с пользователем.
4. **Интеграция**: Приступать к коду и тестам только после явного одобрения.

---

## ⚡ Технические лимиты сервера (VPS ~1GB RAM, 1 vCPU)

* **Ограничение памяти**: Лимит PHP в Docker-контейнере жестко задан в `128M`.
* **Потоковая обработка**: Запрещено загружать большие выборки из БД или файлы в память целиком. Всегда использовать PHP-генераторы (`yield`), курсоры БД и постраничную обработку (chunks).
* **Скрапинг и сеть**: Все сетевые запросы (парсеры, API Telegram) должны иметь жесткие тайм-ауты (timeout <= 10s) во избежание зависания процессов.
* **Однопоточность**: Запрещено использовать многопоточность или запускать тяжелые сборки без ограничений памяти.
