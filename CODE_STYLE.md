# Архитектура и Стиль кода (OOP & SOLID)

В этом проекте действуют строгие правила по написанию чистого, компактного и объектно-ориентированного кода.

## 1. Архитектурные принципы
* **ООП и SOLID**: Код должен быть написан с максимальным использованием принципов SOLID. Следует избегать процедурного программирования (линейных портянок кода). Разделяйте логику по ответственности (Single Responsibility Principle) — если метод берет на себя слишком много, выделяйте логику в небольшие приватные методы-помощники.
* **Лаконичность и отказ от лишних переменных**: Если переменная используется всего один раз, не создавайте ее отдельно. Инлайните выражения прямо в местах использования (например, прямо в условие `if($var = ...)`). Код должен быть максимально чистым и читаться как текст.
* **Паттерны ООП**: Смело задействуйте классические паттерны проектирования (где это уместно), чтобы код был гибким и легко расширяемым.

## 2. Форматирование кода
* **Пробелы**: Не ставьте пробел после управляющих конструкций (`if`, `while`, `foreach`, `catch` и т.д.).
  * *Пример*: `if($condition)`
* **Фигурные скобки**: Открывающая фигурная скобка `{` для тел операторов (включая `try`/`catch`) должна переноситься на новую строку.
  * *Пример*:
    ```php
    if($condition)
    {
        // ...
    }
    ```
* **Длина строки**: Не переносите код на новую строку, если он помещается в одну линию (лимит длины строки — **220 символов**). Делайте переносы только в случае превышения этого лимита.
